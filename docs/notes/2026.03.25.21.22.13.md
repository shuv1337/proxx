That error usually means you’re sending the wrong field name. For GPT-5.4, the documented parameter is `reasoning.summary`, not a top-level `reasoning_summary`, and the examples use the Responses API format `reasoning: { effort: "...", summary: "auto" }`. [github](https://github.com/musistudio/claude-code-router/issues/652)

## Correct shape

Use this request structure instead of:

```json
{
  "model": "gpt-5.4",
  "reasoning_summary": "auto"
}
```

Use:

```json
{
  "model": "gpt-5.4",
  "input": "Your prompt here",
  "reasoning": {
    "effort": "low",
    "summary": "auto"
  }
}
```

OpenAI’s reasoning guide says reasoning summaries are enabled through the `summary` field inside the `reasoning` object, and the example for GPT-5.4 shows exactly that nesting.

## Why this happens

A few SDKs, wrappers, and OpenAI-compatible providers have had bugs or mismatches around reasoning-summary field names, including variants like `reasoningSummary` or `reasoning.generate_summary`, which can trigger `unknown_parameter` errors. [github](https://github.com/anomalyco/opencode/issues/13546)

If you are using:
- A framework wrapper, inspect the raw JSON being sent.
- An OpenAI-compatible provider, check whether it supports GPT-5.4 reasoning summaries yet.
- An older SDK, upgrade it before retrying.

## Practical fix list

1. Switch to the Responses API request format. [github](https://github.com/musistudio/claude-code-router/issues/652)
2. Remove any top-level `reasoning_summary` field.   
3. Put the option under `reasoning.summary`, usually with `"auto"`.   
4. If the error persists, log the final outbound payload, because a wrapper may still be rewriting it to `reasoningSummary` or another unsupported name. [github](https://github.com/openai/openai-go/issues/392)
